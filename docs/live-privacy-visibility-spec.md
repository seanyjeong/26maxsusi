# 수시 라이브 개인정보·가독성 개선 Spec

## Problem

Windows에서 작은 보조 문구가 흐리고, 라이브 화면과 엑셀에 학생 이름·지점·고교가 그대로 노출된다.

## Scope

### In

- Windows에서는 Pretendard 우선, 기본 렌더링과 자간을 사용한다.
- 저대비 보조 색상을 WCAG AA 수준으로 높인다.
- 라이브에 비밀번호 재확인형 개인정보 모드 버튼을 제공한다.
- 개인정보 모드에서 이름·지점·고교를 화면과 엑셀 모두 마스킹한다.
- 학생 고교를 라이브 API 응답과 화면에 포함한다.

### Out

- DB 스키마 변경
- 개인정보 원본 자체의 저장·전송 암호화 변경
- 로그인 방식 변경

## Acceptance Criteria

- 이름 `김민수`는 `김○수`, 지점은 항상 `○○`, 학교 `행복고`는 `행X고`, `서라벌고등학교`는 `서X벌고등학교`로 표시된다.
- 올바른 현재 원장 비밀번호로만 개인정보 모드를 켜거나 끌 수 있다.
- 실패 문구는 상태 코드나 개발 용어 없이 이해하기 쉬운 한국어로 표시된다.
- 개인정보 모드에서는 원문 이름·지점·고교가 화면 DOM과 생성 엑셀 값에 남지 않는다.
- 변경된 런타임 파일은 500줄 이하이고 단위·계약·브라우저 검증이 통과한다.

## API Contract

- 기존 `POST /susi/login`에 현재 JWT의 `userid`와 입력 비밀번호를 보내 재인증한다.
- 기존 `GET /susi/realtime-rank-by-college`의 각 학생에 `학교명`을 추가한다.

## UI Contract

- 상단에 개인정보 모드 버튼을 둔다.
- 버튼을 누르면 비밀번호 확인 모달이 열리고 성공 후 즉시 현재 결과를 다시 그린다.
- 모드는 `sessionStorage`에 탭 단위로 유지한다.

## Test Plan

- 마스킹 순수 함수의 한국어 이름·지점·학교 경계값 단위 테스트
- 비밀번호 재확인 성공·실패 문구와 모드 전환 단위 테스트
- API 학교 필드와 라이브 스크립트 연결 계약 테스트
- Playwright로 버튼·모달·마스킹 화면 확인
